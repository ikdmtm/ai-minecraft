# リポジトリ把握メモ

把握した内容を整理すると、このリポジトリは「AI VTuber が Minecraft ハードコアを自動プレイし、その様子を YouTube 配信する」ための TypeScript 実装です。現行の本番入口は [src/index.ts](/home/ikeda/app/ai-minecraft/src/index.ts) で、起動時はダッシュボードだけ立ち上げ、開始操作を受けるとワールドをリセットし、Minecraft クライアント、VOICEVOX、FFmpeg、YouTube 配信、認知オーケストレータをまとめて起動する構成です。依存関係や全体設計は [ARCHITECTURE.md](/home/ikeda/app/ai-minecraft/ARCHITECTURE.md) にかなり丁寧に書かれています。

中核は `cognitive` 系で、[src/cognitive/orchestrator.ts](/home/ikeda/app/ai-minecraft/src/cognitive/orchestrator.ts) が 3 層を束ねます。[src/cognitive/reflexLayer.ts](/home/ikeda/app/ai-minecraft/src/cognitive/reflexLayer.ts) が実際の Mineflayer 操作主体で、4Hz で危険回避、戦闘、採掘、探索、クラフト、簡易シェルター建設まで行います。[src/cognitive/tacticalLayer.ts](/home/ikeda/app/ai-minecraft/src/cognitive/tacticalLayer.ts) は数秒ごとに LLM で短い実況と短期的な目標修正を返し、[src/cognitive/strategicLayer.ts](/home/ikeda/app/ai-minecraft/src/cognitive/strategicLayer.ts) は 45 秒前後ごとに長期目標とサブゴールを決めます。状態共有は [src/cognitive/sharedState.ts](/home/ikeda/app/ai-minecraft/src/cognitive/sharedState.ts) に集約され、死亡履歴や教訓は [src/cognitive/memory.ts](/home/ikeda/app/ai-minecraft/src/cognitive/memory.ts)、スキルDBは [src/cognitive/skillLibrary.ts](/home/ikeda/app/ai-minecraft/src/cognitive/skillLibrary.ts) が `better-sqlite3` で持っています。

配信まわりは分離がきれいで、VOICEVOX は [src/tts/voicevox.ts](/home/ikeda/app/ai-minecraft/src/tts/voicevox.ts) と [src/tts/audioQueue.ts](/home/ikeda/app/ai-minecraft/src/tts/audioQueue.ts)、映像合成は [src/stream/ffmpeg.ts](/home/ikeda/app/ai-minecraft/src/stream/ffmpeg.ts)、アバター状態は [src/stream/avatar.ts](/home/ikeda/app/ai-minecraft/src/stream/avatar.ts)、表情ファイル更新は [src/stream/avatarRenderer.ts](/home/ikeda/app/ai-minecraft/src/stream/avatarRenderer.ts)、named pipe への RGBA フレーム供給は [src/stream/avatarFrameWriter.ts](/home/ikeda/app/ai-minecraft/src/stream/avatarFrameWriter.ts)、HUD テキスト出力は [src/stream/hudWriter.ts](/home/ikeda/app/ai-minecraft/src/stream/hudWriter.ts) が担当です。YouTube 配信枠の自動作成は [src/youtube/api.ts](/home/ikeda/app/ai-minecraft/src/youtube/api.ts) と [src/youtube/googleApiAdapter.ts](/home/ikeda/app/ai-minecraft/src/youtube/googleApiAdapter.ts) で、OAuth が無ければ固定ストリームキーにフォールバックします。ダッシュボードは [src/dashboard/server.ts](/home/ikeda/app/ai-minecraft/src/dashboard/server.ts)、[src/dashboard/routes.ts](/home/ikeda/app/ai-minecraft/src/dashboard/routes.ts)、[src/dashboard/index.html](/home/ikeda/app/ai-minecraft/src/dashboard/index.html) です。

重要な観察点として、現行本番経路は新しい認知アーキテクチャに一本化されています。[src/index.ts](/home/ikeda/app/ai-minecraft/src/index.ts) が配信ライフサイクルを持ち、[src/runtime/liveStreamSession.ts](/home/ikeda/app/ai-minecraft/src/runtime/liveStreamSession.ts) が配信開始/停止の実務を引き受けます。一方で [src/llm/client.ts](/home/ikeda/app/ai-minecraft/src/llm/client.ts)、[src/bot/client.ts](/home/ikeda/app/ai-minecraft/src/bot/client.ts)、[src/db/repository.ts](/home/ikeda/app/ai-minecraft/src/db/repository.ts)、[src/config/loader.ts](/home/ikeda/app/ai-minecraft/src/config/loader.ts)、[src/health/checker.ts](/home/ikeda/app/ai-minecraft/src/health/checker.ts) には未接続または補助用途の部品が残っています。加えて、HUD は現状 [src/index.ts](/home/ikeda/app/ai-minecraft/src/index.ts) から世代・目標・脅威などは更新されますが、HP・空腹・座標は更新されていないので既定値のままになる可能性があります。`basePosition` も設定している箇所が見当たらず、拠点帰還ロジックはまだ十分に活きていません。

インフラは AWS 前提で、Terraform は [infra/main.tf](/home/ikeda/app/ai-minecraft/infra/main.tf) と [infra/ec2.tf](/home/ikeda/app/ai-minecraft/infra/ec2.tf)、初期構築は [infra/scripts/user-data.sh](/home/ikeda/app/ai-minecraft/infra/scripts/user-data.sh) にまとまっています。ここで Ubuntu 22.04 上に Node 20、FFmpeg、PulseAudio、VOICEVOX Docker、Minecraft server/client、systemd サービスを全部入れる想定です。運用手順は [DEPLOY.md](/home/ikeda/app/ai-minecraft/DEPLOY.md)、本番サービス定義は [infra/scripts/systemd/orchestrator.service](/home/ikeda/app/ai-minecraft/infra/scripts/systemd/orchestrator.service)、デプロイスクリプトは [scripts/deploy.sh](/home/ikeda/app/ai-minecraft/scripts/deploy.sh) にあります。

補足すると、現時点では `npm test -- --runInBand` が通るところまで確認できています。次に手を入れるなら「現行本番経路だけの依存図を作る」「未接続モジュールを整理する」「HUD/拠点管理の未配線を直す」の順がよさそうです。
